# Modelo y controlador en Spring Boot

Si seleccionas líneas de la transcripción en esta sección, irás a la marca de tiempo en el vídeo.

Bueno, ya tenemos la plantilla, pero necesitaremos algo de chicha para rellenarla, así que vamos a seguir avanzando con nuestra aplicación web. 

Empezamos creando el **modelo**, para lo que crearemos el paquete `models` dentro de nuestro paquete.  
New > Package y al nombre que tenemos le añadimos `.models` y ahí dentro creamos una **clase para representar una reunión**. 

¿He dicho reunión? Justo ese es el nombre de la nueva clase.  
New > Class, `Reunion`. Fíjate de meterla dentro del nuevo paquete. E implementamos un **bean clásico** con los atributos:

- `long id`  
- `String asunto`  
- `LocalDate fecha`  

Un par de constructores, el de por defecto y el que recibe todos los atributos, y los **getters y setters**. Esta parte no tiene mucho misterio, se hace como toda la vida, e incluso el propio entorno de desarrollo nos ayuda a hacerla.  

Así que vamos a por el **controlador** y empezamos como antes, creando el paquete `controllers`.  
New > Package, `.controllers`. Y de seguido creamos la clase `ReunionController`.  

Nosotros ya sabemos que es un controlador, pero se lo tenemos que decir a Spring Boot con la anotación `@Controller`. Además, como queremos que ese controlador responda a una ruta concreta, le tenemos que indicar esta ruta con la anotación `@RequestMapping`, a la que le pasaremos entre paréntesis un string con esa ruta: `/reuniones`.  

En una aplicación real, o en una aplicación terminada, como quieras verlo, recuperaríamos la lista de reuniones de alguna fuente de datos, pero de momento, vamos a ponerlos aquí directamente. Así que creamos una **lista de reuniones estática**, vacía. Y para que no todas las reuniones sean simultáneas, le sumamos `i` días a la fecha de ahora.  

Ahora hacemos un método que nos recupere todas las reuniones. Tenemos que etiquetarlo con `@GetMapping` para que responda a las peticiones **GET** que reciba la ruta que hemos indicado antes en el `RequestMapping`.  

Este método debe devolver un **String**. Lo llamaremos `getAllReuniones` y recibe un `Model`, `model`.  
El model que importamos es el del paquete `ui` de `springframework`. Y a ese modelo le añadimos la lista de reuniones como atributo. El nombre que le pongamos al atributo, `reuniones`, debe coincidir con el utilizado en la **vista**.
